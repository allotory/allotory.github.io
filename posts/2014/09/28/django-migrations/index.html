Django 1.7 使用migrations建立数据库连接

周日 28 九月 2014
By Ellery

Django1.7 采用了全新的APP加载框架，校验框架，数据查询优化，并且还自带整合了django-south的（数据库）迁移功能，其提供的django-admin migrate也将代替syncdb命令.

因此在我们建立数据库模型之后，就不能使用django的syncdb命令同步数据库了

python manage.py syncdb

否则数据库模型将不会同步到真正的数据库中。 在Django 1.7中使用将使用全新的migrations同步数据库。建立数据库模型后使用命令

python manage.py makemigrations west

创建migrations，成功显示如下：

Migrations for 'west':
    0001_initial.py:
    - Create model Character

使用sqlmigrate命令查看sql语句

python manage.py sqlmigrate west 0001

最后使用migrate命令同步数据库

python manage.py migrate

此时数据库模型已经真正的同步到数据库中了。
