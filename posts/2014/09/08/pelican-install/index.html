使用Pelican和GitHub搭建个人博客

周一 08 九月 2014
By Ellery

简介

Github是一个具有版本管理功能的代码仓库，每个项目都有一个主页，列出项目的源文件。GitHub设计了Pages功能，允许用户自定义项目首页，用来替代默认的源码列表。所以，github Pages可以被认为是用户编写的、托管在github上的静态网页。

基础知识

搭建博客使用的工具是基于Python的Pelican。相对于WordPress，Pelican有着轻便小巧的特性，与GitHub Pages搭配很完美。当然其他的工具如Jeklly也非常棒，基于octopress相当的不错，可惜都是基于ruby语言的，对ruby语言压根就不了解，并且最近一直在学习Python，所以最终选择了基于Python的Pelican。

  1. 安装Python，本文使用的是Python 2.7
  2. 安装Git，Git简单教程参见Git教程
  3. 将Python安装文件夹中的scripts和make.exe加入环境变量
  4. 安装pip，具体可以看官网介绍。
  5. 安装pelican和markdown，具体请参见markdown教程。
    pip install pelican
pip install markdown

开始搭建博客

打开Git bash，创建一个文件夹Amazilia（可以按喜好拟定），执行如下命令：

mkdir Amazilia
cd Amazilia
pelican-quickstart

执行完pelican-quickstart命令后，会提示博客的配置选项，除少数几项必填之外，其他可以选择默认，如有需要可以在pelicanconf.py文件中可以进行修改。 命令成功执行后，会出现Pelican框架，如下：

Amazilia/
├── content                # 存放输入的markdown源文件
│   └── (pages)            # 存放手工创建的静态页面，可选
├── output                 # 存放最终生成的静态博客
├── develop_server.sh      # 测试服务器
├── Makefile               # 管理博客的Makefile
├── pelicanconf.py         # 配置文件
└── publishconf.py         # 发布文件，可删除

撰写博客

完成博客主体搭建后，可以使用Sublime Text创建一个.md文件，使用markdown语法书写博客，完成后保存于content文件夹中。

###简介
Github是一个具有版本管理功能的代码仓库，每个项目都有一个主页，列出项目的源文件。GitHub设计了Pages功能，允许用户自定义项目首页，用来替代默认的源码列表。所以，github Pages可以被认为是用户编写的、托管在github上的静态网页。

完成之后，在Amazilia目录下，执行make html命令生成博客

make html
(pelican e:/blog/content/ -o e:/blog/output -s e:/blog/pelicanconfg.py)

make html命令是将.md文件生成HTML，存放在output目录下，如果没有make命令，可以使用第二行的Pelican命令。 接着执行make serve命令开启测试服务器

make serve
(cd e:/blog/output/ && python -m pelican.server)

同样make serve命令也可以由第二行的Pelican命令替代，在浏览器中输入http://localhost:8000即可看到博文效果。

配置主题

在Amazilia目录下，按照如下步骤下载Pelican主题，只挑选喜欢的即可，其他不喜欢的可以删掉。主题可以在Pelican主题库查看。

git clone https://github.com/getpelican/pelican-themes.git

打开配置文件pelicanconf.py，更改或添加THEME为自己喜欢的主题。

THEME = 'pelican-themes/gum'

添加评论系统

在Disqus上注册，按照流程Disqus会分配给站点一个Shortname，然后修改pelicanconf.py

DISQUS_SITENAME=Shortname

发布博客

在github上面建立一个当前github用户名.github.io的版本仓库，如我博客版本仓库的为allotory.github.io，然后进入output目录下，依次执行如下命令：

git init
git remote add origin git@github.com:allotory/allotory.github.io.git
git pull origin master
git commit -m 'first blog'
git push origin master

如此完成博客发布，访问allotory.github.io即可。

Google Analytics

注册Google Analytics可以更好的管理自己的站点，按顺序注册，得到Tracking ID配置到pelicanconf.py里面。

GOOGLE_ANALYTICS = 'Tracking ID'

Google Webmasters

Google Webmasters是Google的站长工具，使用它的目的是让博客被Google更好的收录，比如手动让Googlebot抓取、提交Robots、更新Sitemap等功能，使用起来非常方便。要想使用Google Webmasters只要在Google Webmasters上注册即可。

URL配置

使用URL配置可以更好的设置博文的URL。打开pelicanconf.py，按照如下修改或添加：

ARTICLE_URL = 'posts/{date:%Y}/{date:%m}/{date:%d}/{slug}/'
ARTICLE_SAVE_AS = 'posts/{date:%Y}/{date:%m}/{date:%d}/{slug}/index.html'

此时的URL为http://allotory.github.io/posts/2014/09/08/pelican-install/,而文件保存路径为http://allotory.github.io/posts/2014/09/08/pelican-install/index.html。
